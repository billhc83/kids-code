★ Filter default change
filter now defaults to true for guided, full, and free. It defaults to false for open (since open is a free sandbox with no constraints). Use filter:false to override on any other mode, or filter:true on an open step if you want to restrict the palette.
Step 2 — Optional key:value Overrides (applied after preset)
filter:true / filter:false default: true
When true, the palette only shows block types that actually appear in this step's code — nothing extra. When false, the full palette is available. Use filter:false to override on steps where you want a wide-open palette.
fill:true / fill:false default: from guidance
When true, phantom slots are pre-filled with the correct answer (student sees it filled in). When false, slots are empty and the student must drag the right block in. Overrides the guidance preset value.
structure:partial / :full / :none default: from guidance
partial — palette locks to exactly the expected block type when a slot is active.
full — palette shows all section-allowed blocks (no type lock).
none — fully open, no constraints.
validation:step / :none default: from guidance
step — the Complete Step button checks block correctness before advancing.
none — no validation, student can advance freely.
readonly:true / :false default: from guidance
When true, existing placed blocks cannot be selected, moved, or deleted. The student can only interact with empty phantom slots. When false, all blocks are interactive.
view (positional part 3) blocks | editor — default: blocks
Sets the editor interface for this step. blocks shows the drag-and-drop block builder. editor shows the raw code editor. This is positional (3rd pipe segment), not a key:value pair.
Step 3 — Optional Boolean Flags (word alone, anywhere in the header)
reset
Clears all cumulative blocks before this step processes. Use when a step should start completely fresh rather than inheriting all locked blocks from previous steps.
aside
Prevents this step from contributing to the running cumulative block state. Useful for demo or explanation steps that should not leave locked blocks behind for future steps.

Quick reference — all guidance modes + their defaults
guidance structure validation fill filter ★ readonly Typical use
guided (default) partial step false true true Student fills empty typed slots. Palette filtered to needed blocks only.
full full step true true true Slots pre-filled. Student sees complete answer. Still validated.
free none none true true true Complex code given to the student pre-filled, no validation required. Workspace is locked — student reads/observes, doesn't build.
open none none false false false Fully open sandbox. Empty slots, full palette, nothing locked.

★ filter was previously false by default — now true. Only turns off with explicit filter:false.

Minimal forms — what you actually need to write
Two things always default and never need to be written
view blocks is always the default. Only write editor when you specifically want the code editor. Leave it out otherwise.
guidance guided is always the default. Only write a guidance mode when it's something other than guided.
